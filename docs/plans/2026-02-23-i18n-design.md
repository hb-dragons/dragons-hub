# i18n Design: next-intl for Dragons Admin

**Date**: 2026-02-23
**Status**: Approved
**Languages**: German (default) + English
**Library**: next-intl v4.x

## Requirements

- German as default locale, English as secondary
- URL subpath routing (`/admin/matches` for German, `/en/admin/matches` for English) using `as-needed` prefix strategy
- Frontend-only translation — API stays English, frontend maps error codes to translated strings
- SDK data (league names, team names, venues) displayed as-is without translation
- Type-safe translation keys with TypeScript augmentation
- Unified date/time/number formatting (replaces hardcoded `de-DE` calls)
- 100% test coverage maintained

### Core Configuration

**`i18n/routing.ts`**:
```ts
import { defineRouting } from "next-intl/routing";

export const routing = defineRouting({
  locales: ["de", "en"],
  defaultLocale: "de",
  localePrefix: "as-needed",
});
```

**`i18n/request.ts`**:
```ts
import { getRequestConfig } from "next-intl/server";
import { routing } from "./routing";

export default getRequestConfig(async ({ requestLocale }) => {
  let locale = await requestLocale;
  if (!locale || !routing.locales.includes(locale)) {
    locale = routing.defaultLocale;
  }
  return {
    locale,
    messages: (await import(`../messages/${locale}.json`)).default,
    formats: {
      dateTime: {
        matchDate: { weekday: "short", day: "2-digit", month: "2-digit", year: "2-digit" },
        short: { day: "2-digit", month: "2-digit", year: "numeric" },
        syncTimestamp: { month: "short", day: "numeric", hour: "numeric", minute: "2-digit" },
      },
    },
  };
});
```

**`next.config.ts`** addition:
```ts
import createNextIntlPlugin from "next-intl/plugin";
const withNextIntl = createNextIntlPlugin("./src/i18n/request.ts");
export default withNextIntl(nextConfig);
```

**`middleware.ts`**:
```ts
import createMiddleware from "next-intl/middleware";
import { routing } from "./i18n/routing";

export default createMiddleware(routing);

export const config = {
  matcher: ["/((?!_next|api|.*\\..*).*)"],
};
```

## Message Structure

Single flat JSON file per locale using dot-notation keys: `page.component.element`.

**Key categories**:
- `nav.*` — navigation labels
- `matches.*` — match list and detail pages
- `sync.*` — sync dashboard
- `settings.*` — settings pages
- `teams.*` — team management
- `common.*` — shared buttons, labels, states
- `time.*` — relative time expressions
- `errors.*` — error messages mapped from API error codes
- `toast.*` — toast notification messages

**TypeScript augmentation** (`i18n.d.ts`):
```ts
import type messages from "./messages/de.json";

declare module "next-intl" {
  interface AppConfig {
    Messages: typeof messages;
  }
}
```

## Component Patterns

**Server Components** use `getTranslations()`:
```tsx
import { getTranslations } from "next-intl/server";

export default async function MatchesPage() {
  const t = await getTranslations("matches");
  return <h1>{t("title")}</h1>;
}
```

**Client Components** use `useTranslations()`:
```tsx
"use client";
import { useTranslations } from "next-intl";

export function SyncTriggerButton() {
  const t = useTranslations("sync");
  return <Button>{t("trigger")}</Button>;
}
```

## Date/Time Formatting

Replaces all hardcoded `de-DE` and inconsistent formatting with next-intl's `useFormatter()` / `getFormatter()`.

| Current Function | Location | Replacement |
|------------------|----------|-------------|
| `formatDate()` | `lib/format.ts` | `format.dateTime(date, "short")` |
| `formatMatchDate()` | `matches/utils.ts` | `format.dateTime(date, "matchDate")` |
| `formatDate()` | `sync/utils.ts` | `format.dateTime(date, "syncTimestamp")` |
| `formatRelativeTime()` | `sync/utils.ts` | `format.relativeTime()` + message keys |
| Countdown strings | `sync/utils.ts` | Translation keys (`time.inMinutes`, etc.) |

The `@dragons/ui` calendar/date-picker components receive locale via prop:
```tsx
import { useLocale } from "next-intl";
import { de, enUS } from "date-fns/locale";
const localeMap = { de, en: enUS };
```

## Navigation

**`lib/navigation.ts`** re-exports locale-aware utilities:
```ts
import { createNavigation } from "next-intl/navigation";
import { routing } from "@/i18n/routing";

export const { Link, redirect, usePathname, useRouter } = createNavigation(routing);
```

All internal `<Link>` imports switch from `next/link` to `@/lib/navigation`. This auto-prefixes the locale.

## Locale Switcher

A DE/EN toggle placed in the admin header component. Uses `useRouter().replace(pathname, { locale })` to switch without full page reload.

## Middleware Behavior

1. `/admin/matches` → detect locale (cookie > Accept-Language > default `de`) → serve German
2. `/en/admin/matches` → serve English, set locale cookie
3. Unknown locale prefix → redirect to default

## Testing

**Test helper**:
```ts
export function renderWithIntl(ui: React.ReactElement, locale = "de") {
  return render(
    <NextIntlClientProvider locale={locale} messages={messages}>
      {ui}
    </NextIntlClientProvider>
  );
}
```

**Test categories**:
- Middleware: locale detection, redirects, cookie handling
- Components: render correct translated text per locale
- Formatting: date/time output per locale
- Navigation: links include locale prefix
- Locale switcher: URL and content updates
- Message completeness: CI check that `de.json` and `en.json` have identical key sets

## Scope

### In scope
- All ~195 hardcoded UI strings extracted to message files
- Date/time/number formatting unified via next-intl
- URL-based locale routing with middleware
- Locale switcher in admin header
- TypeScript type safety for translation keys
- Tests for all i18n functionality

### Out of scope
- API-side localization (stays English)
- Translating SDK data (league names, team names, venues)
- Additional languages beyond German + English
- Translation management system (TMS) integration
- Better Auth UI localization (external dependency, address separately if needed)
